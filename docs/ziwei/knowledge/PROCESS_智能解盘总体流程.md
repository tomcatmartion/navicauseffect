## 二、完整工作流总览

```
用户输入出生年月日时地
         ↓
【阶段一】生成命盘 + 宫位评分
  ├─ 询问父母生年
  │   ├─ 有父母生年 → 个性化评分（含太岁入卦）
  │   └─ 无父母生年 → 标准评分
  └─ 输出：十二宫评分 + 格局识别 + 三层宫位对照表
         ↓
【阶段二】性格定性 + 事项问诊
  ├─ 输出命主性格图谱与命宫全息底色
  └─ 询问用户关注方向
         ↓
  ┌─────────────────────────────────┐
  │         用户提问方向识别           │
  └─────────────────────────────────┘
         ↓                      ↓
【阶段三】事项分析           【阶段四】互动关系分析
（主线，可随时触发）         （与阶段三并列，可在阶段二后直接触发）
  ├─ 求财分支                ├─ 收集对方生年
  ├─ 求职分支                ├─ 识别关系类型
  ├─ 求爱分支                ├─ 建立虚拟命盘
  ├─ 求学分支                └─ 三维合参输出
  ├─ 求健康分支
  └─ 求名分支
    每个分支均执行：
    问诊聚焦→论原局→论大限→论流年→输出

  ⚡ 阶段三与阶段四可交替触发
     用户可在事项分析中途提及互动关系→切换至阶段四
     互动关系分析完成后可继续回到阶段三
```

---

## 三、各阶段详细说明

### 阶段一：生成命盘 + 宫位评分

**触发：** 用户在网站输入出生年月日时出生地，系统生成斗数命盘数据。

#### Step 1.1 接收命盘数据
- 接收系统生成的命盘JSON数据（包含十二宫星曜、大限、流年信息）
- 调用 `SKILL_原局四化读取规则`：读取生年天干四化 ＋ 太岁宫遁干四化，合并为完整原局四化

#### Step 1.2 询问父母生年（个性化评分）

| 用户回应 | 处理路径 |
|---|---|
| 提供父母生年（任意一方或双方） | 带入父母太岁入卦 → 影响疾厄宫遗传评分 → 调用 SKILL_宫位原生能级评估（含父母入卦版）→ 输出个性化评分报告 |
| 不提供父母生年 | 直接调用 SKILL_宫位原生能级评估（标准版）→ 输出标准评分报告 |

#### Step 1.3 输出评分报告
- 十二宫 Final_Score 评分
- 关键格局识别（大格局 / 护佑机制 / 凶险格局）
- 三层宫位对照表建立（原局 / 大限 / 流年）
- 评分结果存储，供后续阶段调用

---

### 阶段二：性格定性 + 事项问诊

**触发：** 阶段一评分完成后自动触发。

#### Step 2.1 性格定性输出
- 调用 `SKILL_定性命主性格`
- 以命宫 / 身宫 / 太岁宫三宫为核心，四维合参输出完整性格图谱
- 输出命宫全息底色（四化 / 六吉 / 六煞 / 丙丁级），存储供后续分支调用
- 向用户呈现：性格特质、行为模式、核心优势与风险方向

#### Step 2.2 事项问诊识别

| 用户提问方向 | 系统识别 | 进入分支 |
|---|---|---|
| 财运 / 赚钱 / 投资 / 生意 / 合伙 | 求财事项 | 阶段三：求财分支 |
| 工作 / 求职 / 跳槽 / 升职 / 事业 | 求职事项 | 阶段三：求职分支 |
| 感情 / 恋爱 / 婚姻 / 找对象 | 求爱事项 | 阶段三：求爱分支 |
| 读书 / 考试 / 留学 / 升学 | 求学事项 | 阶段三：求学分支 |
| 身体 / 健康 / 疾病 / 体质 | 求健康事项 | 阶段三：求健康分支 |
| 名气 / 成名 / 传播 / 影响力 | 求名事项 | 阶段三：求名分支 |
| 我和XXX的关系 / 相处 / 合作 | 互动关系 | **直接进入阶段四** |
| 整体运势 / 未来趋势 / 现阶段 | 综合评估 | 阶段三：多分支综合输出 |

> ⚡ **重要：** 互动关系不是事项分析的下游，而是与阶段三并列的独立入口。用户在阶段二问诊时直接提及某人关系，即可跳过阶段三直接进入阶段四。

---

### 阶段三：事项分析（主线）

**触发：** 用户明确事项方向后触发。可随时触发，与阶段四交替进行。

#### Step 3.1 事项问诊聚焦
- 调用 `SKILL_求测事项路由`
- 根据事项类型向用户收集前提条件
- 锁定主看宫位与兼看宫位
- 如涉及合伙 / 感情且有对方生年 → 标记需同步调用阶段四

#### Step 3.2 三层合参分析
- 调用 `SKILL_求测事项分析流程`
- 调用 `KB_事项宫位知识库` 取象宫位含义
- 调用 `KB_星曜赋性与事项分类知识库` 取象星曜赋性

| 分析层次 | 分析逻辑 | 核心判断 |
|---|---|---|
| 论原局 | 带入阶段一/二已完成的命宫全息底色；命宫为轴 → 事项宫四维合参 | 命宫强弱决定能否承载事项宫的煞；命宫强→能扛；命宫弱→煞直接穿透 |
| 论大限 | 大限命宫×原局命宫合参 → 定这十年状态；再看大限事项宫状态 | 大限命宫强+事项宫忌煞→忙碌繁多但能扛；大限命宫弱+事项宫忌煞→容易出问题 |
| 论流年 | 流年命宫×大限命宫合参 → 定今年触发点；再看流年事项宫引动 | 流年与大限同向→力量叠加；流年与大限相反→互相制衡 |

#### Step 3.3 综合输出
- 原局底盘定性（强旺 / 中等 / 虚浮 / 凶危）
- 大限激活方向（顺畅期 / 艰辛期 / 危机期 / 转机期）
- 流年引动（推进窗口 / 蛰伏期 / 风险期）
- 综合结论 ＋ 调整建议 / 风险预警

#### Step 3.4 继续追问
- 用户继续提问其他事项 → 重复 Step 3.1-3.3
- 用户问及互动关系 → 切换至阶段四，完成后可返回阶段三

---

### 阶段四：互动关系分析（与阶段三并列）

**触发条件（任意一条均可触发）：**
- 阶段二问诊时用户直接提及与某人的关系 → 直接进入阶段四
- 阶段三事项分析中涉及合伙 / 感情且用户提供对方生年 → 切换至阶段四
- 阶段三完成后用户追问互动关系 → 切换至阶段四

#### Step 4.1 收集对方信息
- 收集对方出生年份
- 确认关系类型：父母 / 子女 / 夫妻情侣 / 合作伙伴 / 同事上司 / 下属
- 无对方生年 → 只做命主单方关系宫分析，告知用户有局限

#### Step 4.2 太岁入卦分析
- 调用 `SKILL_互动关系解读`
- 调用 `SKILL_原局四化读取规则` 读取对方生年四化 ＋ 遁干四化
- 建立虚拟命盘（以对方生年地支为虚拟命宫顺排十二宫）
- 完整入卦：四化 ＋ 禄存羊陀 ＋ 魁钺 ＋ 鸾喜

| 分析维度 | 内容 |
|---|---|
| 入卦者维度 | 对方生年四化（主早/表层）＋ 遁干四化（主晚/深层）＋ 引动星曜 → 对方带着什么心态进入这段关系 |
| 命主维度 | 命主命宫全息底色（已在阶段二完成）→ 命主以什么底色回应对方 |
| 时间维度 | 大限/流年四化对关系宫的引动 → 当下这段关系被激活的方向与时间窗口 |

#### Step 4.3 综合输出
- 互动模式定性（关系画像）
- 核心张力点（1-3个最需注意的方向）
- 可调整建议或不可调整风险预警

## 四、执行禁令

- **严禁跳步执行** —— 必须按阶段一→二→三/四顺序进行，不得跳过任何阶段
- **严禁未完成问诊就开始分析** —— 问诊信息不完整时须继续追问
- **严禁重复做原局分析** —— 原局评分和性格定性只在阶段一二做一次，后续直接调用结果
- **严禁只看事项宫位忽略命宫** —— 每层分析必须先看命宫，再看事项宫
- **严禁将互动关系视为阶段三的下游** —— 互动关系与事项分析并列，可在阶段二后直接触发
- **严禁给出具体金额/时间/事件预测** —— 只判断方向、量级感、时间窗口
- **严禁分析怀孕求子事项** —— 直接礼貌告知此类问题不在分析范围
- **严禁混用宫位** —— 合伙人/上司看迁移宫；下属看仆役宫；不可互换
- **严禁只读生年四化忽略遁干四化** —— 两套四化必须完整读取，地位等同
- **严禁做关系最终结果的绝对判决** —— 只判断互动模式与趋势
- **严禁在不可调整的凶象情况下给出虚假希望** —— 结构性凶象须直接告知风险
