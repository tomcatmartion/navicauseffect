## 二、需要补充的三个缺口

### 缺口 1：丙丁级星曜在性格分析中的缺失

**问题**：`SKILL_定性命主性格.md` 明确要求“丙丁级星曜在命宫时提供辅助性定性”。但当前方案中，`fetchPersonalityTraits` 只处理了主星和六吉六煞，未包含丙丁级（如红鸾、天喜、华盖、天刑、天马、博士十二神等）。

**影响**：用户可能反馈“我命宫有华盖，为什么没提我孤独才艺的一面？”

**解决方案**：扩展知识字典 `star_palace_dict.json`，为丙丁级星曜也建立关键词映射。在 `buildPersonalityIR` 时，遍历命宫/身宫/太岁宫的 `minorStars` 中属于丙丁级的星曜，调用 `fetchPersonalityTraits` 获取关键词，合并到对应层级的 `keywords` 中。

**示例补充**：
```json
"华盖_旺": ["有孤独才艺倾向", "对玄学感兴趣"],
"红鸾_旺": ["感情丰富", "异性缘佳"],
"天刑_旺": ["自律正直"], "天刑_陷": ["容易有官非或手术"],
"天马": ["好动", "喜欢奔波", "适合外出发展"]
```

### 缺口 2：夹宫成立条件与力量对比的代码实现不明确

**问题**：方案中提到“夹宫左右各有星才构成夹”，但未明确在 `getFourDimensions` 或能量加权函数中如何判断“无星”的情况。如果一侧无星，则不应生成“左强右弱”等标签，也不应触发“羊陀夹忌”等特殊格局。

**影响**：可能错误地将单侧有星的情况标记为夹宫失衡。

**解决方案**：在 `getFourDimensions` 中返回 `lin` 对象时，对于空宫（无主星且无辅星），标记 `isEmpty: true`。在能量加权函数中，只有当左右两侧 `isEmpty` 均为 `false` 时，才认为夹宫成立，才进行后续的均衡性判断和特殊夹格检测。如果只有一侧有星，则标注 `lin_gong_status: "单侧有星，无夹"`。

### 缺口 3：身宫与太岁宫的显现时机未在静态分析中提示

**问题**：`SKILL` 文档指出：身宫在第三大限后才逐渐凸显，太岁宫在涉及核心利益/重大抉择时爆发。当前的静态性格 JSON 只给出了三宫标签，但没有提示用户“这些特质可能在不同人生阶段显现”。

**影响**：用户可能误以为身宫特质当下就完全体现，导致对中年后的变化缺乏预期。

**解决方案**：在 JSON 的每个层级中添加 `manifestation_timing` 字段：

```json
{
  "ming_palace": {
    "role": "表层_外在表现",
    "manifestation_timing": "终身显现，早年即凸显",
    ...
  },
  "shen_palace": {
    "role": "中层_行为模式",
    "manifestation_timing": "第三大限后逐渐明显",
    ...
  },
  "tai_sui_palace": {
    "role": "内核_深层动机",
    "manifestation_timing": "涉及核心利益或重大抉择时爆发",
    ...
  }
}
```

同时在大模型的 prompt 中提示：“请根据 manifestation_timing 提示用户不同层面的特质显现时间，避免用户对中年后的变化感到意外。”

## 三、与整体解盘流程的调用设计

### 3.1 被调用的时机

按照 `PROCESS_智能解盘总体流程.md`，性格分析应在**阶段二**完成，即在宫位评分（阶段一）之后、事项问诊（阶段三）之前自动触发。

**当前方案集成方式**：
```typescript
// 在 orchestrator.ts 中，当状态变为 FINAL_IR_READY 后
if (session.state === 'FINAL_IR_READY') {
  const personalityIR = buildPersonalityIR(finalIR);
  const prompt = buildPersonalityPrompt(personalityIR);
  const personalityReport = await aiClient.chat(prompt);
  session.personalityReport = { ir: personalityIR, text: personalityReport };
  session.state = 'PERSONALITY_DONE';
  return personalityReport;
}
```

### 3.2 命宫全息底色的传递

后续事项分析（阶段三）需要全程带入命宫全息底色。我们的方案通过在 `FinalIR` 中存储 `holistic_background`（包含四化焦点、格局、整体调性），并在事项分析的 prompt 中作为“已知背景”注入，实现全程带入。

**示例注入**：
```text
【命主全息底色】
- 生年化忌在父母宫 → 与权威关系处理有执念
- 命宫带遁干忌 → 内心容易纠结
- 格局：机月同梁格 → 适合稳定、辅助性工作
```

### 3.3 对后续分析的约束

在事项分析时，大模型被要求：**所有判断必须结合命主全息底色**，不能脱离性格基础单独论事。这通过在 prompt 中明确指令实现：

> “在分析财运/事业/感情时，请结合命主的性格底色（如内心纠结、外柔内刚等），避免给出与性格矛盾的建议。”
