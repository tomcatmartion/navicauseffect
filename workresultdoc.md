| 2026-06-25 | **修复项目级 CSS Cascade Bug：Tailwind padding/margin 全部失效** | 用户反馈「按钮宽度跟文字几乎一样，需要边距」。Playwright 测量发现**所有 shadcn Button 的 padding-left/right = 0px**，本应 `.px-4`（16px）/`.px-2.5`（10px）等生效。CDP 深查匹配的 CSS 规则发现根因——`ziwei.css:119` 有 unlayered 的 `* { margin:0; padding:0; box-sizing:border-box }`，按 **CSS Cascade Level 5 规范：unlayered rules ALWAYS beat layered rules**（与特异性无关），导致 Tailwind 4 的所有 utilities（`.px-4`/`.py-2`/`.mx-3` 等用 `@layer utilities` 的工具类）被**完全覆盖**，整层 padding/margin 类全部失效。这是项目存在已久的潜在 bug，不仅影响按钮，理论上影响所有 Tailwind padding/margin 工具类。**修复**：用 `@layer base { * { ... } }` 包裹 `ziwei.css:119` 的 reset 规则，让 Tailwind 4 utilities 能正常 cascade 覆盖它。**验证**：Playwright 重测全后台按钮 padding，default=16/16、sm=10/10 全部恢复正常 ✅；analyze_image 视觉确认 payment 页「保存微信/支付宝配置」、models 页「添加模型/编辑/设为默认」按钮左右内边距 16-24px，呼吸感充足；首页/价格/登录等前台页面**无视觉破坏**。`pnpm tsc --noEmit` 零错误。**教训追加 imemory E-67**：Tailwind 4 项目里所有 reset/base 性质的 CSS 规则必须放进 `@layer base`，否则会 silently 覆盖所有 Tailwind utilities。 | 修复 |
